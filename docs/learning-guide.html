1. What is an AI Gateway? (Start Here)

Real-World Analogy: The Hotel Front Desk

Imagine a hotel with 10 departments (housekeeping, restaurant, spa, etc.). Each department needs to order supplies from different vendors (Amazon, Walmart, Costco). Without a gateway: each department orders directly — nobody tracks total spending, each has their own account, and there's no bulk discount. With a gateway: ALL orders go through the front desk. The front desk tracks every order, enforces budgets ("housekeeping can only spend $500/month"), gets bulk pricing, and produces monthly reports. That front desk is your AI Gateway.

First, What is an LLM?

An LLM (Large Language Model) is an AI that reads text and writes text back. Think of it as a very smart auto-complete. Companies like OpenAI (GPT-4), Anthropic (Claude), and AWS (Bedrock) run these models on massive servers and let you use them via the internet. You pay per "token" — a token is roughly 3/4 of a word.

What is a Token? ═══════════════ "Hello, how are you today?" ──┬── ──┬─ ──┬─ ──┬─ ──┬── 1 2 3 4 5 = 5 tokens (roughly) Rule of thumb: ┌──────────────────────────────────────┐ │ 1 token ≈ 4 characters ≈ 0.75 words │ │ 100 tokens ≈ 75 words ≈ 1 paragraph │ │ 1,000 tokens ≈ 750 words ≈ 1.5 pages │ └──────────────────────────────────────┘ You pay SEPARATELY for: ┌─────────────┐ ┌──────────────┐ │ INPUT tokens │ and │ OUTPUT tokens │ │ (your prompt)│ │ (AI's reply) │ │ CHEAPER │ │ EXPENSIVE │ └─────────────┘ └──────────────┘

# Your app calls OpenAI directly
curl https://api.openai.com/v1/chat/completions \
  -H "Authorization: Bearer sk-your-openai-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-4o",
    "messages": [
      {"role": "system", "content": "You are a helpful assistant."},
      {"role": "user", "content": "What is Kubernetes?"}
    ],
    "temperature": 0.7,
    "max_tokens": 200
  }'

# Response:
{
  "id": "chatcmpl-abc123",
  "model": "gpt-4o",
  "choices": [{
    "message": {
      "role": "assistant",
      "content": "Kubernetes is an open-source container orchestration platform..."
    },
    "finish_reason": "stop"
  }],
  "usage": {

That's it. You sent 25 tokens in, got 87 tokens out. At GPT-4o pricing ($2.50/1M input, $10/1M output), that cost: (25 × $2.50 + 87 × $10) / 1,000,000 = $0.000933 — less than a penny.

Analogy: The Airport Security Checkpoint

Every passenger (request) goes through security (the gateway). The Magic: ONE Line of Code Changes

The gateway speaks the exact same language as OpenAI. Teams don't rewrite their code. They change ONE line:

BEFORE (direct to OpenAI)

from openai import OpenAI

client = OpenAI(
    api_key="sk-your-openai-key"
)
response = client.chat.completions.create(
    model="gpt-4o",
    messages=[{"role": "user", "content": "Hi"}]
)
→

AFTER (through gateway)

from openai import OpenAI

client = OpenAI(
    api_key="gw-team-api-key-here",
    base_url="http://gateway:8000/v1"  # ONLY CHANGE!
)
response = client.chat.completions.create(
    model="gpt-4o",
    messages=[{"role": "user", "content": "Hi"}]
)

Same library, same code, same everything. Just a different base_url and a gateway-issued API key.

What changed? 1 Authentication — "Who are you?"

Analogy: Hotel Key Card

When you swipe your hotel card, the system doesn't compare it to every card ever made. It reads the code, looks it up in the database, and knows your room number, checkout date, and which doors you can open. Our API key works the same way.

2 Rate Limiting — "Are you going too fast?"

Analogy: Theme Park Ride

A roller coaster can only take 60 people per hour. If 61st person arrives, they wait. But unlike a fixed "per hour" rule, we use a sliding window — we always look at the last 60 seconds, not fixed minute boundaries. More on this in Section 8.

Sliding Window Rate Limiting (limit = 5 per minute) ════════════════════════════════════════════════════ Timeline (each ■ = one request): 10:00:00 10:00:15 10:00:30 10:00:45 10:01:00 ─────┬─────────┬──────────┬──────────┬──────────┬──────► ■ ■ ■ ■■ │ 1st 2nd 3rd 4th,5th │ │ At 10:00:58 ──► "Can I make a 6th request?" │ Window looks back 60 seconds from NOW: │ ◄──────────── 60 seconds ────────────────────── NOW [10:00:01] [10:00:15] [10:00:30] [10:00:45] [10:00:55] ■ ■ ■ ■ ■ Count = 5/5 DENIED! At limit. At 10:01:02 ──► Window slides forward... ◄──────────── 60 seconds ──────────────────── NOW [10:00:15] [10:00:30] [10:00:45] [10:00:55] ■ ■ ■ ■ 10:00:01 fell out! Count = 4/5 ALLOWED! Space freed up.

4 Cache Check — "Have we answered this before?"

Analogy: Google's Cached Results

If 100 people Google "weather today", Google doesn't recalculate from scratch each time. It caches the answer. Our gateway does the same: if 100 users ask "How do I reset my password?" with the same settings, we call OpenAI ONCE and serve the cached answer 99 times. But only when temperature=0 (deterministic), because with temperature > 0, the AI intentionally gives different answers each time.

5 Provider Routing — "Which AI provider should handle this?"

Analogy: Travel Booking Site

When you search for a flight, Kayak checks United, Delta, Southwest, etc. If United's website is down, it still shows you Delta's flights. Our gateway does the same: it knows which AI providers support which models, and if one is down, it automatically tries the next one.

@router.post("/v1/chat/completions")
async def chat_completions(
    request: ChatCompletionRequest,           # Pydantic validates the JSON body
    authorization: str = Header(...),          # Extract Authorization header
    db: AsyncSession = Depends(get_db),        # Database session (injected)
    redis: Redis = Depends(get_redis),         # Redis client (injected)
):
    request_id = str(uuid.uuid4())[:8]
    start_time = time.time()

    # ── STEP 1: AUTH ─────────────────────────────────────────────
    # Extract "Bearer gw-xxx" → hash → look up in DB
    token = authorization.replace("Bearer ", "")
    key_hash = hashlib.sha256(token.encode()).hexdigest()

    api_key = await db.execute(
        select(ApiKey)
        .options(joinedload(ApiKey.team))
        .where(ApiKey.key_hash == key_hash, ApiKey.is_active == True)
    )
    api_key = api_key.scalar_one_or_none()
    if not api_key:
        return JSONResponse(status_code=401, content={"error": "Invalid API key"})

    team = api_key.team

    # ── STEP 2: RATE LIMIT ──────────────────────────────────────
    rate_result = await check_rate_limit(redis, str(team.id), team.rate_limit_rpm)
    if not rate_result.allowed:
        return JSONResponse(
            status_code=429,
            content={"error": f"Rate limit exceeded. Retry in {rate_result.reset_at:.0f}s"},
            headers={"Retry-After": str(int(rate_result.reset_at - time.time()))}
        )

    # ── STEP 3: POLICY ──────────────────────────────────────────
    decision = await evaluate_policies(db, str(team.id), request)
    if not decision.allowed:
        return JSONResponse(
            status_code=403,
            content={"error": f"Blocked by policy: {decision.reason}"}
        )

    # ── STEP 4: CACHE CHECK ─────────────────────────────────────
    cached = await get_cached_response(redis, request)
    if cached:
        cached["gateway_metadata"] = {
            "cost_usd": 0.0,           # No cost — it was cached!
            "cache_hit": True,
            "provider": "cache",
            "latency_ms": int((time.time() - start_time) * 1000),
            "request_id": request_id,
        }
        return JSONResponse(content=cached)

    # ── STEP 5: ROUTE TO PROVIDER ────────────────────────────────
    try:
        response = await provider_router.route_request(request)
        provider_name = provider_router.get_provider_name_for_model(request.model)
    except RuntimeError as e:
        return JSONResponse(status_code=503, content={"error": str(e)})

    # ── STEP 6: CALCULATE COST ───────────────────────────────────
    cost = calculate_cost(
        request.model,
        response.usage.prompt_tokens,
        response.usage.completion_tokens,
    )

    # ── STEP 7: AUDIT LOG ───────────────────────────────────────
    await audit.log_request(
        db, request_id, team.id, api_key.id,
        model=request.model, provider=provider_name,
        prompt_tokens=response.usage.prompt_tokens,
        completion_tokens=response.usage.completion_tokens,
        cost_usd=cost,
        latency_ms=int((time.time() - start_time) * 1000),
        status="success",
    )

    # ── STEP 8: USAGE AGGREGATION ────────────────────────────────
    await update_usage_summary(
        db, str(team.id), request.model, provider_name,
        response.usage.prompt_tokens, response.usage.completion_tokens,
        response.usage.total_tokens, cost,
    )

    # ── CACHE + RETURN ───────────────────────────────────────────
    response_dict = response.model_dump()
    await set_cached_response(redis, request, response_dict)
    response_dict["gateway_metadata"] = {
        "cost_usd": cost,
        "cache_hit": False,
        "provider": provider_name,
        "latency_ms": int((time.time() - start_time) * 1000),
        "request_id": request_id,
    }
    return JSONResponse(content=response_dict)

def _get_pricing(model: str) -> tuple[float, float]:
    if model in MODEL_PRICING:                    # Exact match
        return MODEL_PRICING[model]
    for known, pricing in MODEL_PRICING.items():  # Prefix match
        if model.startswith(known):
            return pricing
    return (10.00, 30.00)                          # Default (conservative)

Fail-open: Rate limiter allows all. Cache always misses. Why? Blocking ALL traffic because Redis is down is worse than allowing extra traffic. Auth (PostgreSQL) still works.

Fail-closed: Auth fails, all requests return 500. Why? Without auth, we can't know who's making requests. Fix: Cache API keys in Redis with 5-min TTL.

21. Interview Questions & Answers (25+)

Architecture & System Design

Q1: Walk me through what happens when a request hits your gateway.
A: 8-step pipeline: (1) Auth — SHA256 hash, DB lookup. (2) Rate limit — Redis sorted set, 60s sliding window. (3) Policy — fnmatch glob matching, priority-sorted. (4) Cache — temp=0 only, SHA256 key. (5) Route — provider preference order with failover. (6) Cost — per-model pricing with prefix matching. (7) Audit — INSERT. (8) Usage — UPSERT daily summary.
If deeper: Gateway overhead ~12ms. LLM takes 500ms-5s. <2% latency added.
Q2: Why FastAPI over Flask or Django?
A: (1) Async native — handles 100s of concurrent LLM waits. (2) Pydantic — validation + serialization built-in. (3) Performance — one of fastest Python frameworks.
"Why not Go?": I/O bound, not CPU bound. Python async is fine. Team familiarity.
Q3: Scale to 10,000 RPS?
A: Stateless gateway → horizontal scale. Redis handles 100K+ ops/sec. Bottleneck: PostgreSQL audit → decouple via Kafka, batch-insert. Partition audit_logs monthly. Read replica for dashboard. PgBouncer for connections.
Q4: Why sliding window over token bucket?
A: Token bucket allows bursts. LLM providers reject bursts with 429s. Sliding window enforces even distribution. Redis sorted set, O(log N), max 60 entries per team.
Q5: Why SHA256 instead of bcrypt?
A: API keys are 64+ random chars (high entropy) — brute force impossible. SHA256: 0.001ms. bcrypt: 100ms. Same approach as Stripe, GitHub, AWS.
Q6: How does the policy engine handle conflicts?
A: Priority-based. Sort DESC, first match wins. No match = default ALLOW.
Same priority? Undefined order. Use distinct priorities. Could add deny-wins tiebreaker.
Q7: What if Redis goes down?
A: Fail-open. Rate limit allows all, cache always misses. Blocking traffic is worse. Auth/audit (PostgreSQL) still work.
Q8: Why pre-aggregate usage?
A: "Cost by team for March" — audit_logs scans 30K rows. usage_summaries reads ~50 rows. Fast dashboards.

Implementation

Q9: Bedrock adapter vs OpenAI?
A: (1) Auth: IAM SigV4 vs Bearer. (2) Format: Converse API translation. (3) Async: boto3 sync → asyncio.to_thread().
Q10: Adding streaming?
A: StreamingResponse (SSE) when stream=true. Forward chunks immediately. Token counts in final chunk for audit. Cache harder — buffer full response first.
Q11: Crash after LLM, before audit?
A: Write "pending" before LLM call, update to "completed" after. Stuck >60s = lost. Monthly reconciliation.
Q12: Why async SQLAlchemy?
A: Sync blocks the event loop. asyncpg = non-blocking, purpose-built for async PostgreSQL.

Database & Security

Q13: Partition audit_logs?
A: Range partition by created_at, monthly. Old partitions: detach, dump to S3, drop. Create ahead via cron.
Q14: Why UUIDs?
A: Don't reveal counts. Client-side generation. Safe for distributed writes.
Q15: PostgreSQL vs MongoDB?
A: Relational data, JOINs, upserts, JSONB for flexibility, asyncpg speed, range partitioning.
Q16: Security measures?
A: SHA256 hashing, admin isolation, team isolation, body truncation, env secrets, Docker network isolation, gw- prefix scanning.
Q17: Key rotation?
A: Multiple active keys per team. Create new, deploy, verify old unused via last_used_at, revoke old. Zero downtime.

Tradeoffs & Philosophy

Q18: What tradeoffs?
A: (1) Fail-open rate limiting — availability. (2) Sync audit — simpler, +10ms. (3) No streaming — simpler cost tracking. (4) temp=0 cache — safe, lower hits. (5) Pre-aggregated usage — fast reads, write amplification.
Q19: Improvements with more time?
A: Streaming (SSE), circuit breakers, Kafka audit, hard budgets, webhooks, smart routing, multi-region.
Q20: Why OpenAI-compatible?
A: One config change. Every LLM tool (LangChain, LlamaIndex) supports custom base_url. Industry standard.
Q21: vs LiteLLM / Portkey?
A: SaaS gateways = your traffic through their servers (privacy). LiteLLM = routing only. Ours = self-hosted, policy engine, budget management, enterprise governance.
Q22: Observability?
A: Three pillars: structured JSON logs with request_id, Prometheus metrics, OpenTelemetry traces. Alerts on cost, errors, latency.
Q23: Budget enforcement?
A: Check usage_summaries for month. Cache in Redis (60s TTL). Over budget → 429. Can overshoot by up to 60s.
Q24: Skills demonstrated?
A: System design, API design, data modeling, distributed systems, security, testing, DevOps, full-stack.
Q25: Prompt injection?
A: Gateway is routing layer, not content. Could add content filter service, policy conditions with content rules, suspicious request flagging.
